Job Scanner

GitHub ↗

Job Scanner is a personal automation that monitors company career pages across four source types, scores every role against a custom fit rubric using an LLM, and pushes anything qualifying directly into a Notion database — filtered, ranked, and ready to act on. It runs every 2 hours via GitHub Actions at no cost.

The goal was to stop manually checking job boards and instead have a pipeline that does the work continuously: ingest roles, apply consistent scoring criteria, and surface only the leads worth looking at.

Source coverage

The scanner pulls from four distinct source types to maximize coverage without relying on any single aggregator:

Company boards Explicit Greenhouse, Lever, and Ashby slugs defined in a private config — monitors specific companies directly at the source, bypassing aggregator lag
VC portfolio discovery Probes job boards for companies sourced from top VC portfolio pages without needing slugs in advance — the scanner derives and tests slug candidates automatically
Aggregators Broad searches via Adzuna and The Muse across all companies, catching roles that don't appear on direct boards or portfolio pages
YC Work at a Startup Dedicated scan of YC-backed companies for matching roles, covering a large slice of early-stage companies in a single pass

How it works

On each run the scanner fetches open roles from all four source types, filters for NYC / hybrid / remote-friendly positions above a salary floor, scores each role 1–10, and pushes anything at or above the threshold into Notion automatically.

Scoring runs across several independent signals: title match against tiered keyword lists, description keywords that signal the kind of work environment I want, company signals like stage and known investors, and penalties for corporate language, quota-carrying sales roles, experience requirements above a threshold, or off-target locations. Roles below 6.0 are dropped entirely. Roles between 6.0 and 6.4 land in Notion flagged for manual review. Roles at 6.5 and above are pushed as qualified leads.

Configuration

The targeting logic lives in two gitignored config files — companies.py and config.py — so the public repo contains the full pipeline without exposing the specific companies or scoring weights I use. The main levers are keyword tiers for title matching, role DNA keywords that signal the right kind of work, penalty terms that disqualify poor fits, and VC portfolio company lists the scanner uses to find boards automatically.

Deployment

GitHub Actions Recommended deployment — runs every 2 hours automatically on the free tier. Config and company lists are injected as repo secrets, keeping targeting private
Render Alternative deployment for ~$1–3/month if you prefer a persistent service; the repo includes a render.yaml that deploys automatically on push

Stack

Python Core scanner — fetches, filters, scores, and pushes roles in a single script with no framework overhead
Notion API Output layer — qualified roles are written directly to a Roles database with structured properties for score, source, location, and review flag
Adzuna + The Muse Aggregator sources — free API keys cover broad company searches that complement the direct board monitoring
GitHub Actions Scheduled execution — cron job runs every 2 hours with secrets injected at runtime; no server required
