The Problem

Music theory education has a fundamental gap: the tools that teach it don’t connect to real music, and the tools that play real music don’t teach anything. I identified a space where no product existed — and where the technical components to build one had only recently become accessible.

What I Built

I owned the full scope — product definition, system architecture, and technical direction — managing the build end-to-end. The goal was to integrate multiple AI models into a single coherent system, process audio in real time, and deliver a frictionless interface entirely in the browser. No downloads, no setup, with results surfacing before the full pipeline completes.

How I Built It

I selected and integrated three AI models to handle distinct parts of the problem: Meta’s Demucs for audio separation running on cloud GPU, Spotify’s Basic Pitch for converting audio into analyzable note data, and Essentia for key and tempo detection. A language model (Claude) translates the harmonic output into plain-English context. Coordinating these systems — managing sequencing, latency, and partial result delivery — was the core engineering challenge.

I made a deliberate decision to run all playback client-side via the Web Audio API, keeping the server out of the critical path for user interaction. A concurrent job queue handles simultaneous requests and streams results progressively, so the interface stays responsive well before the full pipeline finishes.

The Outcome

Riffd is live at riffdlabs.com — a production system built and shipped, integrating three AI models across a full stack from cloud GPU to browser-native interface. It scales on demand and operates at near-zero infrastructure cost outside of processing time.

Stack

Demucs Meta’s AI model for audio separation — splits a song into up to 6 stems on cloud GPU with a second melodic pass
Basic Pitch Spotify’s pitch detection model — converts each separated part into note events for chord analysis
Essentia Key and tempo detection directly from the audio signal
Web Audio API Browser-based mixing engine — all playback, muting, and pitch control runs client-side with no server roundtrips
Claude API Generates plain-English theory context and chord progression explanations
Flask + Vanilla JS Lightweight Python backend, zero-dependency frontend. Concurrent job queue, deployed on Render
