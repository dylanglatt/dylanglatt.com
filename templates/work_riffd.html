Riffd is a music analysis tool built for musicians who want to understand what's happening inside a track — not just listen to it. Give it a YouTube URL or an audio file and it returns separated stems, generated tabs, chord analysis, and production credits.

The core use case is simple: you want to learn a song, figure out what key it's in, understand what the bass is doing, or find out who produced it. Riffd handles all of that in one place, without requiring any music theory background to use.

How it works

Audio separation runs on Demucs via Replicate's GPU API — offloading the heavy ML workload rather than grinding on a shared CPU. Tab generation uses Basic Pitch for pitch detection on each stem. Chord analysis and key detection run against the separated harmonic content. Production credits come from MusicBrainz.

The backend is a Flask app deployed on Render, with a job queue that lets users track long-running separations in real time. The full pipeline — download, separate, analyze, credit — runs asynchronously and streams status back to the frontend as it progresses.

Why I built it

I wanted a single tool that could do stem separation, tab output, and credits lookup without switching between four different services. I also wanted to explore building on top of modern audio ML — Demucs and Basic Pitch together produce genuinely useful results that weren't accessible to regular musicians a few years ago.
