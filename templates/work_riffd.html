The Problem

If you want to understand how a song works — what key it’s in, what chords are being played, how the bass line moves under the melody — you need a trained ear and a theory background. Existing tools either teach theory in the abstract or just play music. Nothing took a real song and broke it down for you.

What I Built

A tool where you pick any song and get it taken apart. Riffd separates the track into individual instruments (vocals, bass, drums, guitar, piano, other), detects the key and tempo, identifies the chords, and explains the theory behind the progression — all in the browser, no downloads, no setup.

How It Works

Three applied AI models handle the heavy lifting. Demucs (Meta) runs on a cloud GPU and splits the audio into isolated stems. Basic Pitch (Spotify) converts those stems into note events so I can extract chords. Essentia detects the key and tempo from the raw signal. An LLM takes all of that output and writes a plain-English explanation of the theory.

The hard part was orchestration. Each model has different latency, and users shouldn’t wait for all of them to finish before seeing anything. I built an automated processing pipeline that streams results progressively — stems arrive first, then harmonic analysis, then the theory write-up. All playback runs client-side via the Web Audio API, so the server is never in the loop after processing. The Python backend coordinates five API integrations (Spotify, Claude, Genius, Last.fm, and YouTube) alongside the ML models.

The Result

Live at riffdlabs.com. A zero-to-one build: I scoped the product, designed the architecture, built every layer of the stack, and handled deployment.

Stack

Demucs Meta’s AI model for audio separation — splits a song into up to 6 stems on cloud GPU with a second melodic pass
Basic Pitch Spotify’s pitch detection model — converts each separated part into note events for chord analysis
Essentia Key and tempo detection directly from the audio signal
Web Audio API Browser-based mixing engine — all playback, muting, and pitch control runs client-side with no server roundtrips
Claude API Generates plain-English theory context and chord progression explanations
Flask + Vanilla JS Lightweight Python backend, zero-dependency frontend. Concurrent job queue, deployed on Render
