Argus

argusosint.vercel.app ↗ GitHub ↗

The Problem

Open-source conflict data is abundant but nearly unusable raw. GDELT ingests hundreds of thousands of events per day — and most of them are noise: agricultural burns, local crime, diplomatic statements, and protest coverage all land in the same stream as actual armed conflict. By the time genuine escalation is visible in the news, it’s already been packaged into narrative. I wanted a system that could cut to signal before any of that happened.

What I Set Out to Build

A full OSINT pipeline modeled after how defense and intelligence analysts actually work — not a dashboard with a map pasted on top, but a system with real noise filtering, satellite corroboration, escalation detection, and a dense operational interface designed for pattern recognition across multiple simultaneous theaters.

How It Works

The pipeline starts with live GDELT 2.0 ingestion every 15 minutes: downloading and parsing conflict event exports, normalizing CAMEO codes into actionable categories (battles, explosions, violence against civilians, riots, strategic developments), and scoring each event by impact using an inverted Goldstein scale.

A two-tier noise filter follows. High-confidence kinetic events — armed actor types, extreme Goldstein scores, multi-outlet coverage — auto-pass. Everything else is routed to Claude Haiku, which fetches the underlying article and evaluates it against strict criteria: confirmed weapons use, casualties, or territorial effect. Crime, diplomacy, protests, and opinion pieces are rejected.

For battles and explosions, satellite corroboration is available via NASA FIRMS VIIRS thermal anomaly data. Events are spatially matched against a historical conflict zone index built from POLECAT — a CIA/DARPA-funded dataset — to suppress false positives from agricultural burns and gas flares. Corroborated events display a “SAT” badge and fire radiative power readings in the detail panel.

The frontend is a Blueprint-dark analyst workstation: a Mapbox GL map with clustered markers sized by media coverage volume; a scrollable event feed sortable by impact score; a 6-hour UTC time-series chart of event frequency by type; and an escalation detection system that surfaces regions with ≥30% 24-hour event increases. A Country Brief panel pulls live UN OCHA/ReliefWeb humanitarian reports for any selected region. Analysts can confirm or dismiss events, with feedback persisted to Vercel Blob.

A separate Python backtesting pipeline cross-references GDELT against POLECAT across 2023–2024, measuring hotspot agreement, intensity correlation, and source divergence — providing empirical grounding for the system’s signal quality claims.

The Result

Argus filters a raw 100K+ event/day stream down to a clean, high-confidence conflict feed, corroborated where possible by satellite imagery and validated against a decade of historical conflict data. The operational interface surfaces what’s actually escalating — with the sourcing and corroboration to back it up — before it becomes a headline.

Data Sources

GDELT 2.0 Open public event database updated every 15 minutes — hundreds of thousands of events/day across 100+ countries, sourced from global media
POLECAT CIA/DARPA-funded political event dataset — used to build the historical conflict zone index for satellite corroboration and false positive suppression
NASA FIRMS VIIRS Near-real-time satellite thermal anomaly data — spatially matched to battle and explosion events for independent corroboration
UN OCHA / ReliefWeb Live humanitarian situation reports pulled per country — surfaced in the Country Brief panel for selected regions
CAMEO + Goldstein Scale Standardized event classification and severity scoring — CAMEO codes normalized into actionable categories, Goldstein inverted for impact ranking

Stack

React 19 + Mapbox GL JS Frontend analyst workstation — clustered event map, sortable feed, 6-hour time-series chart, and escalation detection UI
Express.js API layer serving ingested events, corroboration data, and analyst feedback endpoints
Claude Haiku (Anthropic API) Second-tier noise filter — evaluates fetched article excerpts against strict kinetic event criteria, rejecting crime, diplomacy, and non-conflict content
Python data pipeline Scheduled GDELT ingestion, normalization, satellite matching, and 2023–2024 backtesting against POLECAT
Vercel + Vercel Blob Deployment and analyst feedback persistence
