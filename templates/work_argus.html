Argus

In development

Argus is an OSINT dashboard built to track global conflict events in near-real time. It ingests from GDELT 2.0 — an open event database covering hundreds of thousands of events per day across 100+ countries, updated every 15 minutes — and surfaces escalation patterns as they develop in the data, before they've been filtered through editorial judgment and into headlines.

The core interface is a severity-ranked event feed with a live map overlay, backed by Goldstein Scale scoring and CAMEO event codes to normalize events across actors, regions, and conflict types. Trend charts show escalation and de-escalation patterns over time by geography and category.

Why I built it

Following geopolitical events through news alone is slow and noisy — it rewards the most alarming headlines, not the most structurally significant signals. I wanted a tool that pulls raw event data before it's been filtered through editorial judgment, normalizes it against a consistent scoring system, and lets me build my own read of what's happening and where.

GDELT makes this tractable. The dataset covers hundreds of thousands of events per day, encoded against a rigorous ontology. Argus is essentially a layer on top of that — an analytical interface tuned to the questions I actually want to answer.

How it works

A scheduled job pulls GDELT 2.0 exports every 15 minutes, parses and normalizes the TSV data, and writes structured event records to a local database. Each event is tagged with its CAMEO action code, actors, country codes, and Goldstein Scale score — a continuous measure of conflict intensity ranging from −10 (most conflictual) to +10 (most cooperative).

The dashboard aggregates these records into a severity-ranked feed filtered by recency, geography, and event type. The map view plots events geospatially with intensity encoding. Trend views surface rolling escalation curves at the country and region level, making it easy to spot when a situation is moving fast.

Data & Analytical Framework

GDELT 2.0 Open event database updated every 15 minutes, covering hundreds of thousands of events per day encoded against a standardized ontology of actors, actions, and locations
CAMEO Codes Conflict and Mediation Event Observations ontology — standardized classification of events across actor types and action categories, enabling cross-region normalization
Goldstein Scale Continuous conflict severity score from −10 (most conflictual) to +10 (most cooperative), applied per event for severity ranking and trend detection

Stack

Python + Pandas Scheduled ingestion pipeline — TSV parsing, normalization, deduplication, and aggregation on every 15-minute GDELT export
Mapbox Geospatial event map with intensity encoding — cluster size and color reflect Goldstein scores and event density per region, with live refresh
Flask + SQLite Python backend with Jinja2 templates and vanilla JS — structured event storage, query layer, and dashboard UI served from a single lightweight app
